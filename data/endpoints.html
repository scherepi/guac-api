Endpoint List:

  • GET -
  • GET /api/hello - returns a simple hello world, but everyone has to start somewhere :)
  • GET /api/djb/{CODE} - returns a DJB hash for the string provided
  • GET /api/journal - returns the latest communal journal entry
  • GET /api/journal/{USER} - returns the latest journal entry for user USER.
  • POST /api/journal - takes application/json data, writes to the communal journal whatever's in the "entry" field.
  • POST /api/journal/register/{USER} - registers a new journal for the user providede! (note: does not take data, just make a raw POST request)
  • POST /api/journal/{USER} - writes a new journal entry for the registered user USER. data works the same as /api/journal/
